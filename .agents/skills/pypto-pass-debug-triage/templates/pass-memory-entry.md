# Pass 设计记忆文档模板

> 用途：沉淀单个 Pass 的设计意图、不变量、脆弱点与历史决策，降低人员流动带来的知识流失。
> 生成方式：由 agent 读取 Pass 源码 + 相关 bug 模式 + git 历史后按此模板产出。

---

## 1. 基本信息

| 字段 | 内容 |
|------|------|
| Pass 名称 | {PassName} |
| 所属目录 | `framework/src/passes/{sub_dir}/` |
| 主要源文件 | `{file1}.cpp`、`{file2}.h` |
| Pipeline 阶段 | {block_graph/tensor_graph/tile_graph/...} |
| 前置依赖 Pass | {list} |
| 后置消费 Pass | {list} |
| 对应 bug 模式 | {Sxxx/Pxxx/Nxxx} |
