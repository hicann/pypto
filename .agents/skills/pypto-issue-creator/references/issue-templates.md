# Issue 模板

本文档定义了5种Issue类型的标准模板。

---

## 目录

1. [Bug Report](#1-bug-report)
2. [Feature Request](#2-feature-request)
3. [Documentation](#3-documentation)
4. [Question](#4-question)
5. [Task](#5-task)

---

## 1. Bug Report

**触发场景**: 报错、异常、失败、精度问题、性能问题

**必填字段**: 标题、现象描述、复现步骤、期望行为

```markdown
## Bug 描述

[清晰简洁地描述问题]

## 复现步骤

1. [步骤1]
2. [步骤2]
3. [步骤3]

## 期望行为

[描述应该发生什么]

## 实际行为

[描述实际发生了什么]

## 环境信息

- **CANN 版本**: [自动填充]
- **PyPTO Commit**: [自动填充]
- **服务器类型**: [A2/A3]
- **Python 版本**: [自动填充]
- **操作系统**: [自动填充]

## 日志/报错信息

```
[粘贴相关日志或报错信息]
```

## 最小复现代码

```python
[提供最小复现代码]
```

## 补充信息

[任何其他有助于理解问题的信息]
```

---

## 2. Feature Request

**触发场景**: 希望支持、新增功能、优化、扩展

**必填字段**: 标题、功能描述、使用场景

```markdown
## 功能描述

[清晰描述你希望添加的功能]

## 使用场景

[描述这个功能将解决什么问题，适用于什么场景]

## 期望的 API/用法

```python
[描述你期望的API或使用方式]
```

## 动机

[为什么需要这个功能？它带来什么价值？]

## 参考实现

- **PyTorch**: [相关文档/实现链接]
- **Triton**: [相关文档/实现链接]
- **cuTile**: [相关文档/实现链接]
- **TileLang**: [相关文档/实现链接]

## 补充信息

[任何其他有助于理解需求的信息]
```

---

## 3. Documentation

**触发场景**: 文档缺失、文档错误、文档改进

**必填字段**: 标题、问题描述、影响范围

```markdown
## 文档问题描述

[描述文档存在的问题]

## 影响范围

- [ ] API 文档
- [ ] 使用教程
- [ ] 示例代码
- [ ] 其他: [说明]

## 当前文档内容

[粘贴当前的文档内容，如有]

## 建议的修改

[描述应该如何修改，或提供修改后的内容]

## 相关链接

[提供相关文档或代码的链接]

## 补充信息

[任何其他有助于理解问题的信息]
```

---

## 4. Question

**触发场景**: 如何使用、为什么、不理解

**必填字段**: 标题、问题描述

```markdown
## 问题描述

[清晰描述你的问题]

## 上下文

[提供问题的背景信息]

## 尝试过的方法

[描述你已经尝试过的方法]

## 相关文档/代码

[提供相关的文档链接或代码片段]

## 补充信息

[任何其他有助于理解问题的信息]
```

---

## 5. Task

**触发场景**: 需要开发、计划、任务

**必填字段**: 标题、任务描述、验收标准

```markdown
## 任务描述

[清晰描述需要完成的任务]

## 背景

[提供任务的背景信息]

## 具体要求

1. [要求1]
2. [要求2]
3. [要求3]

## 验收标准

- [ ] [标准1]
- [ ] [标准2]
- [ ] [标准3]

## 时间要求

[描述任务的时间要求，如有]

## 相关资源

[提供相关的文档、代码或资源链接]

## 补充信息

[任何其他有助于理解任务的信息]
```

---

## 标签映射

| Issue 类型 | 默认标签 |
|-----------|---------|
| Bug Report | `bug`, `needs-triage` |
| Feature Request | `enhancement`, `needs-discussion` |
| Documentation | `documentation` |
| Question | `question` |
| Task | `task`, `needs-assignment` |
